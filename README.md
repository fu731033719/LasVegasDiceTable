# Las Vegas Dice Table

一个基于 `Three.js + Vite + Trystero` 的拉斯维加斯桌面骰子游戏 Web 版本，支持本地多人和局域网联机游玩。

## 游戏特色

- 2 到 6 人对局
- 3D 桌面、3D 骰子、3D 硬币表现
- 支持局域网联机房间
- 支持玩家占座、改名、同步操作
- 支持小游戏 `浑水摸鱼`、`猜正反`、`猜大小`
- 支持重摇筹码和最终金额结算

## 游戏玩法

### 基础流程

1. 每位玩家开局拥有 `8` 枚基础六面骰子，以及 `2` 个重摇筹码。
2. 轮到当前玩家时，点击桌面中央六边形区域进行投掷。
3. 玩家需要从当前投出的骰子中选择一个点数，并将该点数对应的全部骰子一次性放入对应放置区。
4. 若对本次投掷结果不满意，可以消耗 `1` 个重摇筹码，对当前剩余骰子重新投掷一次。
5. 当所有玩家手中的骰子全部放完后，进入最终结算。

### 放置区规则

- 棋盘共有 `6` 个放置区。
- `1 / 2 / 3` 点为功能区，会触发小游戏。
- `4 / 5 / 6` 点为基础游戏区，不触发额外流程。
- 每个区域只认“数量唯一”的对象。
- 如果两个或以上对象在同一区域内骰子数量相同，则这些相同数量全部作废。
- 最终按有效数量从高到低决定第一名、第二名奖励。

| 点数区域 | 区域名称 | 第一名 | 第二名 |
| --- | --- | ---: | ---: |
| 1 | 浑水摸鱼 | 40000 | 40000 |
| 2 | 猜正反 | 50000 | 30000 |
| 3 | 猜大小 | 50000 | 30000 |
| 4 | 基础区 | 90000 | 40000 |
| 5 | 基础区 | 80000 | 30000 |
| 6 | 基础区 | 50000 | 50000 |

## 小游戏规则

### 1 点：浑水摸鱼

- 小游戏内存在一组可消耗的污染骰子堆，初始数量为：

```text
[2, 2, 3, 1, 1, 2]
```

- 玩家可拖动其中一堆污染骰子放到任意区域。
- 所有污染骰子都归属于虚拟对象 `干扰骰子`。
- `干扰骰子` 只参与“重复数量判定”，不会参与最终奖励排名。
- 每一堆污染骰子在本局中只能使用一次，用掉后不会再次出现。

### 2 点：猜正反

- 玩家可以猜测硬币结果为 `正` 或 `反`。
- 最多可进行 `3` 次。
- 每猜中一次获得 `20000`。
- 中途猜错则本小游戏累计奖励清零。
- 玩家可以随时选择结算离场。

### 3 点：猜大小

- 小游戏开始时会先投掷 `2` 枚骰子，得到初始基准点数。
- 之后每回合可以猜：
  - 更大
  - 相等
  - 更小
- 猜中即可进入下一关，也可以随时结算离场。
- 奖池顺序为：

```text
[5000, 8000, 15000, 30000, 50000, 80000]
```

### 重摇筹码

- 每位玩家初始拥有 `2` 个重摇筹码。
- 每消耗 `1` 个筹码，可在一次投掷结束后重新投掷当前剩余骰子。
- 对局结束时，每个未使用的重摇筹码可兑换 `10000`。

## 联机说明

- 房主可创建房间，并通过 URL 中的房间标识邀请其他玩家加入。
- 访客可自主选择座位、修改自己的玩家名称，并进行对应回合操作。
- 当前联机方案适合局域网或支持 HTTPS 的访问环境。

## 本地启动

### 安装依赖

```bash
npm install
```

### 开发模式

```bash
npm run dev
```

或使用局域网访问：

```bash
npm run dev:host
```

默认访问地址通常为：

```text
http://localhost:5173
```

### HTTPS 开发模式

```bash
npm run dev:https
```

### 生产构建

```bash
npm run build
```

### 本地预览构建结果

```bash
npm run preview
```
